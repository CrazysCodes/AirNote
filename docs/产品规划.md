# 极简浮窗备忘录 / 临时收纳口袋产品规划

## 产品定位 checklist

- [x] 明确产品不是知识库，不承载长期知识管理、双链、层级目录、全文写作等复杂能力。
- [x] 明确产品不是复杂 Todo，不追求项目管理、日历排期、多人协作、提醒矩阵。
- [x] 明确产品不是 AI 写作工具，AI 只作为低干扰整理辅助。
- [x] 明确核心价值是“3 秒打开、3 秒记录、3 秒复制”。
- [x] 明确首个场景是微信浮窗 / 手机浏览器中的轻量网页工具。
- [x] 明确差异点是微信场景顺手、常用内容一键复制、临时内容自动沉底。
- [ ] 验证“微信浮窗”是否能稳定保留登录态、页面状态和输入草稿。
- [ ] 确认产品名称、域名、图标和一句话描述。

## 目标用户 checklist

- [x] 微信文件传输助手重度用户，经常把临时信息发给自己。
- [x] 经常记录链接、想法、待办、地址、账号备注、常用话术、常用 prompt 的用户。
- [x] 不想打开复杂 App，只想把内容先塞进去的用户。
- [x] 有少量高频复制内容，例如地址、联系方式、发票抬头、固定回复的人。
- [x] 先服务自用和相似用户，不强行定义为大众笔记产品。
- [ ] 访谈 3-5 个同类用户，确认他们是否真的愿意把文件传输助手中的部分内容迁移出来。
- [ ] 记录用户最常复制的 10 类内容，反推常用区默认模板。

## 核心痛点 checklist

- [x] 文件传输助手内容按消息流混杂，查找成本高。
- [x] 临时内容和长期内容混在一起，过期信息长期占据视线。
- [x] 常用信息每次都要翻聊天记录或备忘录。
- [x] 系统备忘录虽然可用，但不一定在微信聊天场景里顺手。
- [x] 用户不想分类、不想维护、不想进入复杂编辑器。
- [x] 很多内容只有几小时到几天价值，需要自然沉底或清理。
- [ ] 量化关键体验指标：打开到可输入小于 3 秒，保存后反馈小于 0.5 秒，复制一步完成。

## MVP 范围 checklist

- [x] 移动端优先的单页网页应用。
- [x] 首屏包含快速输入框、保存按钮、快捷标签和常用内容固定区。
- [x] 支持文本、链接、简单 checklist 文本保存。
- [x] 支持快捷标签：今天、本周、长期、常用。
- [x] 支持常用内容置顶、复制、隐藏敏感内容。
- [x] 支持临时内容按时间倒序查看。
- [x] 支持筛选：今天、最近 7 天、长期。
- [x] 支持内容过期标记和一键清理已过期内容。
- [ ] 暂不做复杂富文本、图片附件、多人协作、公开分享、文件上传。
- [ ] 暂不做强提醒系统，避免滑向复杂 Todo。
- [ ] 暂不做多级分类，避免制造维护负担。

## 核心用户流程 checklist

- [x] 记录流程：微信浮窗打开页面 -> 光标自动聚焦输入框 -> 输入或粘贴内容 -> 选择快捷标签 -> 保存 -> 回到聊天。
- [x] 复制流程：打开页面 -> 常用区点击目标内容 -> 自动复制 -> 显示轻量 toast -> 回到聊天粘贴。
- [x] 临时清理流程：打开页面 -> 查看过期提示 -> 点击清理 -> 已过期内容进入归档或删除。
- [x] 常用沉淀流程：临时内容中选择“设为常用” -> 补充标题或保留自动标题 -> 进入常用区。
- [x] 敏感内容流程：保存常用内容时开启隐藏 -> 默认只显示标题和掩码 -> 点击后短暂显示或直接复制。
- [ ] 定义误删恢复流程：删除后 5-10 秒撤销，还是提供回收站。

## 页面清单 checklist

- [x] 首页 / 口袋页：输入、快捷标签、常用固定区、临时内容区集中在一页。
- [x] 常用内容管理抽屉：新增、编辑、置顶、隐藏敏感内容。
- [x] 内容详情 / 编辑轻弹层：编辑文本、标签、过期策略、设为常用。
- [x] 清理确认弹层：展示将清理的数量和范围，提供取消。
- [ ] 设置页：账号、数据备份、默认过期时间、隐私说明。
- [ ] AI 整理入口：只在选中文本或详情中出现，不进入主导航。

## 功能模块 checklist

- [x] 快速输入模块：输入框、粘贴识别、保存、保存后清空。
- [x] 快捷标签模块：今天、本周、长期、常用。
- [x] 常用内容模块：标题、正文、复制、置顶、隐藏敏感、编辑、删除。
- [x] 临时内容模块：倒序列表、标签筛选、过期状态、设为常用、删除。
- [x] 轻量搜索模块：按关键词搜索常用和临时内容。
- [x] 清理模块：识别过期内容、一键清理、撤销。
- [ ] 账号模块：匿名本地模式、手机号 / 邮箱登录或微信内登录待确认。
- [ ] 同步模块：单设备本地优先还是云端同步待确认。
- [ ] 导出模块：导出 Markdown / JSON，降低迁移顾虑。

## 数据结构 checklist

- [x] NoteItem 字段：id、content、type、tag、createdAt、updatedAt、expiresAt、isPinned、isArchived、isSensitive、source。
- [x] CommonItem 字段：id、title、content、category、isPinned、isSensitive、copyCount、lastCopiedAt、createdAt、updatedAt。
- [x] Tag 枚举：today、week、longTerm、common。
- [x] ContentType 枚举：text、link、checklist、address、account、prompt、reply、unknown。
- [x] UserPreference 字段：defaultTag、defaultExpireDays、sensitiveRevealSeconds、autoFocusInput、themeMode。
- [x] AuditEvent 字段：id、action、targetType、targetId、createdAt，用于本地调试或轻量行为分析。
- [ ] 确认敏感内容是否需要本地加密存储。
- [ ] 确认已清理内容是软删除、归档还是永久删除。

## API 设计 checklist

- [x] `POST /api/notes`：新增临时内容或长期内容。
- [x] `GET /api/notes?range=today|7d|longTerm&keyword=`：查询临时内容。
- [x] `PATCH /api/notes/:id`：更新内容、标签、过期时间、敏感状态。
- [x] `DELETE /api/notes/:id`：删除单条内容。
- [x] `POST /api/notes/:id/promote`：将临时内容设为常用。
- [x] `GET /api/common-items`：查询常用内容。
- [x] `POST /api/common-items`：新增常用内容。
- [x] `PATCH /api/common-items/:id`：编辑常用内容、置顶、隐藏敏感。
- [x] `POST /api/common-items/:id/copy-events`：记录复制事件，用于排序和自用统计。
- [x] `POST /api/cleanup/expired`：清理过期内容。
- [ ] 确认 MVP 是否先使用浏览器本地存储，暂不提供后端 API。
- [ ] 确认后端 API 是否需要登录鉴权、频控和 CSRF 防护。

## AI 能力 checklist

- [x] AI 只做辅助，不进入主流程强依赖。
- [x] 自动识别内容类型：待办、链接、地址、想法、账号备注。
- [x] 将一段乱文本整理为 checklist。
- [x] 每晚生成“今天别忘了”摘要。
- [x] AI 结果必须可撤销、可忽略，不自动覆盖原文。
- [ ] MVP 默认不接 AI，先用规则识别链接和 checklist。
- [ ] 待确认 AI 调用触发方式：手动点击、保存后后台建议、每日定时摘要。
- [ ] 待确认 AI 成本上限：每人每日调用次数、摘要长度、模型选择。
- [ ] 风险：AI 入口过多会让产品变复杂；避免方式是只在详情页或批量整理时出现。

## 前端实现 checklist

- [x] 移动端优先，适配微信浏览器、Safari、Chrome。
- [x] 首页首屏不做营销内容，直接进入工具。
- [x] 输入框自动聚焦，并保留未保存草稿。
- [x] 保存、复制、清理操作提供轻量 toast。
- [x] 使用底部轻操作区或抽屉承载编辑，不做复杂页面跳转。
- [x] 常用区使用紧凑卡片或列表，支持一键复制。
- [x] 临时区默认倒序，过期内容视觉沉底。
- [x] 使用 PWA 能力作为可选增强：添加到主屏幕、离线缓存。
- [ ] 确认技术栈：Next.js / React SPA / Vue / 原生轻量实现。
- [ ] 确认是否使用 Tailwind CSS 或现有设计系统。
- [ ] 验证微信浏览器中的剪贴板 API 兼容性和降级方案。

## 后端实现 checklist

- [x] MVP 可选方案 A：纯前端 + IndexedDB / localStorage，本地优先，最快验证。
- [x] MVP 可选方案 B：轻量后端 + SQLite / Postgres，支持跨设备同步。
- [x] 后端需要提供内容 CRUD、常用内容 CRUD、清理过期内容、用户偏好。
- [x] 敏感内容需要考虑服务端加密、传输 HTTPS、日志脱敏。
- [x] 清理任务需要支持手动触发，后续再考虑定时任务。
- [ ] 推荐默认：先做本地优先 MVP，待确认跨设备需求后再加后端同步。
- [ ] 待确认部署方式：Vercel / Cloudflare Pages / 自有服务器 / 微信内可访问域名。
- [ ] 待确认身份体系：无登录、本地 PIN、邮箱登录、微信登录。

## 技术方案 checklist

- [x] 阶段 1 推荐采用本地优先架构：React 或 Next.js 静态页面 + IndexedDB。
- [x] 本地优先优点：开发快、隐私风险低、无需账号即可使用。
- [x] 本地优先缺点：换设备不方便，清缓存有丢数据风险。
- [x] 阶段 2 增加同步后端：REST API + SQLite/Postgres + 用户登录。
- [x] 阶段 2 增加导入导出：JSON / Markdown 备份。
- [x] 阶段 3 增加 AI 服务：内容识别、checklist 整理、每日摘要。
- [ ] 待确认首个实现是否接受“单设备自用”作为 MVP 约束。
- [ ] 待确认是否必须支持微信授权登录。

## 设计与交互 checklist

- [x] 视觉关键词：极简、紧凑、安静、工具感、低维护成本。
- [x] 首屏布局：顶部状态与搜索、快速输入、快捷标签、常用区、临时区。
- [x] 输入区必须比其他元素更醒目，但不能像长文编辑器。
- [x] 常用内容优先展示 3-6 条高频项，一屏内可复制。
- [x] 临时内容以信息密度和可扫读为优先，不做大卡片堆叠。
- [x] 敏感内容默认掩码，复制不一定需要明文展示。
- [x] 空状态应该直接引导输入，不讲产品理念。
- [x] 按钮和操作尽量使用图标 + 短标签，避免说明文字占据主屏。
- [ ] 确认品牌气质：更像微信内工具、iOS 备忘录、还是独立极简产品。
- [ ] 验证单手操作区域：保存、复制、清理是否方便拇指触达。

## 测试与验收 checklist

- [x] 打开页面后 3 秒内可以开始输入。
- [x] 保存一条文本内容不超过 2 次点击。
- [x] 复制常用内容只需要 1 次点击。
- [x] 临时内容默认按时间倒序显示。
- [x] 今天、最近 7 天、长期筛选结果正确。
- [x] 已过期内容可以一键清理，并可撤销。
- [x] 敏感内容默认隐藏，复制成功后不泄露完整内容到界面。
- [x] 微信浏览器中复制、输入、返回浮窗后状态可用。
- [x] 弱网或离线时，本地保存不丢失。
- [ ] 覆盖 IndexedDB / localStorage 存储异常场景。
- [ ] 覆盖移动端小屏、安全区、键盘弹起后的布局。
- [ ] 覆盖数据导出和恢复流程。

## 风险与避免方式 checklist

- [x] 风险：和系统备忘录差异不足；避免方式：围绕微信浮窗、一键复制、自动沉底打透。
- [x] 风险：功能膨胀成复杂笔记；避免方式：不做目录、富文本、知识库、协作。
- [x] 风险：常用区变成又一个混乱列表；避免方式：限制默认展示数量，按置顶和复制频次排序。
- [x] 风险：AI 变成鸡肋或成本黑洞；避免方式：MVP 不强依赖 AI，只提供手动触发能力。
- [x] 风险：敏感信息泄露；避免方式：默认本地优先、掩码展示、日志脱敏、后续加密。
- [x] 风险：清理误删；避免方式：清理前展示数量，清理后提供撤销或短期归档。
- [x] 风险：微信浏览器剪贴板限制；避免方式：验证 Clipboard API，失败时显示手动复制选中态。

## 待确认决策 checklist

- [ ] 产品首个版本是否坚持本地优先、无登录、单设备可用。
- [ ] 是否必须支持跨设备同步；如果必须，V1 就需要后端和账号体系。
- [ ] 敏感内容是否需要加密存储；如果需要，需要确认密钥来自用户 PIN 还是设备能力。
- [ ] 过期内容默认策略：今天标签次日过期、本周标签 7 天后过期，还是只沉底不删除。
- [ ] 删除策略：永久删除、软删除、短期回收站三选一。
- [ ] 常用内容是否需要分组；推荐 V1 不做分组，只做置顶和搜索。
- [ ] 是否做微信登录；推荐 V1 不做，除非同步是刚需。
- [ ] AI 是否进入 V1.1；推荐等核心手动流程稳定后再接入。
- [ ] 首个技术栈选择：React SPA、Next.js、Vue 或原生轻量实现。
- [ ] 首个部署目标：本地自用、Vercel、Cloudflare Pages、自有服务器。
