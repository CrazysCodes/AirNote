# 极简浮窗备忘录 / 临时收纳口袋实现清单

## 阶段 0：项目准备 checklist

- [ ] 初始化前端项目：Vite + React + TypeScript。
- [ ] 配置代码规范：ESLint、Prettier、TypeScript strict。
- [ ] 配置样式方案：Tailwind CSS 或 CSS Modules。
- [ ] 建立目录结构：`src/components`、`src/services`、`src/models`、`src/hooks`、`src/styles`。
- [ ] 建立基础移动端 viewport、safe-area、根字体和颜色变量。
- [ ] 增加中文 README，说明本地启动、构建、测试方式。

## 阶段 1：基础 UI checklist

- [ ] 实现移动端应用壳：顶部标题、搜索图标、设置图标。
- [ ] 实现快速输入框：固定高度、可多行、键盘友好。
- [ ] 实现保存按钮：空内容禁用、保存中状态、保存成功 toast。
- [ ] 实现快捷标签：今天、本周、长期、常用。
- [ ] 实现常用固定区：紧凑列表、复制按钮、敏感掩码。
- [ ] 实现临时内容筛选 tabs：今天、最近 7 天、长期。
- [ ] 实现临时内容列表：类型标签、时间、摘要、更多操作。
- [ ] 实现底部清理入口：显示过期数量。
- [ ] 验收：首屏不是营销页，打开后核心输入和复制入口可见。

## 阶段 2：本地数据 checklist

- [ ] 定义 `NoteItem`、`CommonItem`、`UserPreference` TypeScript 类型。
- [ ] 引入或封装 IndexedDB 访问层。
- [ ] 实现 `createNote`。

- [ ] 实现 `updateNote`。
- [ ] 实现 `deleteNote`。

- [ ] 实现 `createCommonItem`。

## 阶段 3：核心交互 checklist

- [ ] 保存临时内容后自动清空输入框。
- [ ] 根据快捷标签自动设置过期时间。
- [ ] 链接内容自动识别为 link。
- [ ] 简单 checklist 文本自动识别为 checklist。
- [ ] 临时内容按 `createdAt` 倒序。
- [ ] 过期内容在列表中沉底或弱化展示。
- [ ] 支持把临时内容设为常用。
- [ ] 支持复制常用内容并记录复制次数。
- [ ] 复制失败时显示手动复制 fallback。
- [ ] 验收：记录、保存、复制三个动作均能在 1-2 次点击内完成。
