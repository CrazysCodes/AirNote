# 极简浮窗备忘录 / 临时收纳口袋技术方案

## 技术目标 checklist

- [x] 支持移动端网页，优先适配微信浏览器和手机浏览器。
- [x] 首屏加载快，打开后可以立即输入。
- [x] 保存和复制链路尽量不依赖网络。
- [x] MVP 避免账号体系和复杂后端，优先验证自用核心体验。
- [x] 后续可平滑扩展到云同步、登录、AI 整理。
- [x] 敏感内容默认最小化暴露，日志和调试信息不得输出明文敏感内容。

## 推荐架构 checklist

- [x] V1 推荐：本地优先单页应用。
- [x] 前端：React SPA 或 Next.js 静态导出。
- [x] 样式：Tailwind CSS 或轻量 CSS Modules，保持组件尺寸稳定。
- [x] 状态：Zustand / React Context 均可，优先选择轻量方案。
- [x] 存储：IndexedDB 为主，localStorage 仅保存轻量偏好和草稿。
- [x] 构建：Vite + React 是最快 MVP 组合。
- [ ] 待确认是否使用 Next.js；如果后续很快要做 API 和登录，Next.js 更省迁移成本。
- [ ] 待确认是否接受单设备本地数据；如果不接受，需要 V1 引入后端。

## 前端模块 checklist

- [x] `QuickCapture`：快速输入、快捷标签、保存。
- [x] `PinnedCommonList`：常用内容展示、复制、敏感掩码、置顶。
- [x] `TempNoteList`：临时内容倒序、筛选、过期沉底。
- [x] `NoteEditorSheet`：轻量编辑、设为常用、删除。
- [x] `CommonItemSheet`：新增/编辑常用内容、敏感开关。
- [x] `CleanupBar`：显示过期数量、一键清理、撤销。
- [x] `Toast`：保存、复制、清理结果反馈。
- [x] `SettingsSheet`：默认过期时间、导出、隐私说明。
- [ ] 微信浏览器剪贴板降级：复制失败时自动选中文本并提示长按复制。
- [ ] 键盘弹起布局：输入区和保存按钮不能被遮挡。

## 本地数据存储 checklist

- [x] IndexedDB 数据库名：`pocket_memo_db`。
- [x] 表 `notes`：存储临时和长期内容。
- [x] 表 `common_items`：存储常用固定内容。
- [x] 表 `preferences`：存储默认标签、默认过期时间、敏感展示时长。
- [x] 表 `trash`：可选，用于短期撤销和误删恢复。
- [x] 草稿 key：`pocket_memo_draft`，使用 localStorage 即可。
- [ ] 确认是否引入 `idb` 库封装 IndexedDB，推荐使用，减少浏览器兼容细节。
- [ ] 确认敏感内容是否加密；V1 可先掩码显示，V1.1 再做 PIN 派生密钥加密。

## 数据模型 checklist

- [x] `NoteItem.id`：字符串 UUID。
- [x] `NoteItem.content`：原文内容。
- [x] `NoteItem.type`：`text | link | checklist | address | account | prompt | reply | unknown`。
- [x] `NoteItem.tag`：`today | week | longTerm | common`。
- [x] `NoteItem.expiresAt`：可为空；为空表示不自动过期。
- [x] `NoteItem.isArchived`：清理后软归档。
- [x] `NoteItem.isSensitive`：是否默认掩码。
- [x] `CommonItem.copyCount` 和 `lastCopiedAt`：用于高频排序。
- [x] `UserPreference.defaultExpireDays`：默认过期天数。

## API 方案 checklist

- [x] V1 本地优先时，不需要真实远程 API。
- [x] 在前端创建 `StorageService`，方法命名对齐未来 API，方便迁移。
- [x] `createNote(payload)` 对齐未来 `POST /api/notes`。
- [x] `listNotes(filter)` 对齐未来 `GET /api/notes`。
- [x] `updateNote(id, patch)` 对齐未来 `PATCH /api/notes/:id`。
- [x] `deleteNote(id)` 对齐未来 `DELETE /api/notes/:id`。
- [x] `promoteToCommon(noteId)` 对齐未来 `POST /api/notes/:id/promote`。
- [x] `recordCopy(commonItemId)` 对齐未来 `POST /api/common-items/:id/copy-events`。
- [x] `cleanupExpired()` 对齐未来 `POST /api/cleanup/expired`。
- [ ] 如果 V1 引入后端，需要补鉴权、CSRF、防重放、接口频控。

## 后端扩展方案 checklist

- [x] V1.1 / V2 可增加轻量后端。
- [x] 推荐技术：Next.js Route Handlers 或 Node.js + Fastify。
- [x] 数据库：SQLite 适合自用部署，Postgres 适合多人和云同步。
- [x] ORM：Drizzle 或 Prisma，推荐 Drizzle，轻量且迁移清晰。
- [x] 鉴权：邮箱验证码、Passkey、微信登录三选一。
- [x] 同步策略：以 `updatedAt` 和 `deletedAt` 做增量同步。
- [x] 冲突策略：同一内容多端编辑时保留最新版本，并记录本地冲突副本。
- [ ] 待确认是否需要微信登录；这是后端复杂度的最大开关之一。

## 隐私与安全 checklist

- [x] 默认不上传用户内容，降低隐私风险。
- [x] 敏感内容列表中只显示标题和掩码。
- [x] 复制敏感内容时可以不展示明文，只提示复制成功。
- [x] 不在日志中记录完整内容、手机号、地址、账号、发票信息。
- [x] 数据导出需要用户主动触发。
- [ ] V1.1 增加本地 PIN：进入应用或显示敏感内容前校验。
- [ ] V1.1 增加敏感内容本地加密：使用用户 PIN 派生密钥。
- [ ] 云同步版本必须明确隐私承诺、删除策略和数据保留周期。

## AI 能力接入 checklist

- [x] V1 不接 AI，先用规则能力验证核心体验。
- [x] 规则识别链接：URL 正则。
- [x] 规则识别 checklist：多行 `- `、`1.`、`[ ]`。
- [x] 规则识别可能敏感：手机号、身份证、邮箱、银行卡样式仅做本地提示。
- [ ] V1.1 手动整理：用户点击“整理为清单”才调用 AI。
- [ ] V2 每日摘要：每天固定时间汇总今天未处理内容。
- [ ] AI 成本控制：限制每日次数、限制输入长度、提供本地规则 fallback。
- [ ] AI 隐私策略：调用前明确提示内容会发送到模型服务。

## 部署方案 checklist

- [x] 自用 MVP：静态网页部署到 Vercel / Cloudflare Pages。
- [x] 本地测试：`npm run dev`，手机通过局域网访问验证微信浏览器表现。
- [x] PWA：可添加到主屏幕，但微信浮窗仍是主入口。
- [ ] 如果使用微信登录，需要配置可访问域名、HTTPS、备案和微信开放平台能力。
- [ ] 如果使用云同步，需要补数据库备份和导出能力。

## 技术风险 checklist

- [x] 微信浏览器 Clipboard API 兼容性不稳定；需要降级手动复制。
- [x] IndexedDB 可能被清缓存删除；需要提供导出备份。
- [x] 自动聚焦可能被移动浏览器限制；需要点击输入区兜底。
- [x] PWA 在微信内体验有限；不能把 PWA 当核心卖点。
- [x] 后端同步会显著增加账号、隐私、成本复杂度；不建议 V1 默认引入。
- [x] AI 会增加成本和隐私压力；应在核心流程稳定后再接入。

## 推荐默认决策 checklist

- [x] V1 使用 Vite + React + TypeScript。
- [x] V1 使用 IndexedDB + localStorage。
- [x] V1 无登录、无云同步。
- [x] V1 不接 AI。
- [x] V1 支持导出 JSON，降低本地数据丢失焦虑。
- [x] V1 默认过期策略：今天 1 天、本周 7 天、长期不过期、常用不过期。
- [x] V1 删除策略：先软删除到短期撤销区，清理后 10 秒可撤销。
